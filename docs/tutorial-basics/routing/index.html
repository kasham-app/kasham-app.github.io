routing

Routing in JET, the magnificent framework, begins by creating & defining routes in the routes directory in your application root. The command below will generate a route file in your application /routes directory in the root directory named web.php

Create Routes
php jet jet:generate-routes web

These routes are responsible for mapping URLs to specific actions in your application. Fear not, for I shall guide you through the enchanting world of routing.

At its core, a JET route consists of a URI and a Closure. This Closure is where the magic happens. Let's start with the most basic routes:

$router->get('jet', function () {
    return 'Hello jet!!';
});

$router->post('jet', function () {
    // Your code here
});

These routes respond to the GET and POST HTTP verbs, respectively, and are associated with the URI /foo.

But behold, for JET offers a diverse array of router methods to handle any HTTP verb that comes your way:

$router->get($JetUrl, $JetCallback);
$router->post($JetUrl, $JetCallback);
$router->put($JetUrl, $JetCallback);
$router->patch($JetUrl, $JetCallback);
$router->delete($JetUrl, $JetCallback);
$router->options($JetUrl, $JetCallback);

Now, let us venture into the realm of route parameters. As you traverse the intricate paths of your application, you may need to capture segments of the URI. Fear not, for JET grants you the power to define route parameters by encasing them within "curly" braces:

$router->get('user/{id}', function ($id) {
    return 'User '.$id;
});

These route parameters shall be passed into your route's Closure when the route is executed, allowing you to access and utilize them in your code.

But heed this warning: route parameters cannot contain the "-" character. Instead, embrace the power of the underscore (_) to represent them.

In your noble quest, you may encounter the need for optional route parameters. To achieve this, you can enclose a part of the route URI definition within square brackets ([...]). For example, "/foo[bar]" will match both "/foo" and "/foobar". Note that optional parameters are only supported in a trailing position of the URI:

$router->get('user[/{name}]', function ($name = null) {
    return $name;
});

In your pursuit of perfection, you may wish to impose constraints on the format of your route parameters. JET understands your desires and allows you to define regular expression constraints within your route definitions:

$router->get('user/{name:[A-Za-z]+}', function ($name) {
    // Your code here
});

As you delve deeper into the art of routing, you shall discover the power of named routes. By assigning a name to a specific route, you unlock the ability to generate URLs or redirects with ease. To bestow a name upon a route, use the as key in the route definition:

$router->get('profile', ['as' => 'profile', function () {
    // Your code here
}]);

Even the paths to controller actions can bear names:

$router->get('profile', [
    'as' => 'profile', 'uses' => 'UserController@showProfile'
]);

Once a route has been graced with a name, you can generate URLs or redirects to it using the mighty route function:

// Generating URLs...
$url = route('profile');

// Generating Redirects...
return redirect()->route('profile');

If the named route defines parameters, you can pass the parameters as the second argument to the route function. These parameters will automatically be inserted into the URL in their correct positions:



$router->get('user/{id}/profile', ['as' => 'profile', function ($id) {
    // Your code here
}]);

$url = route('profile', ['id' => 1]);

Now, let us delve into the wonders of route groups. With route groups, you can share attributes such as middleware or namespaces across multiple routes without repeating them. Behold the power of route groups!

Middleware can be assigned to all routes within a group using the middleware key in the group attribute array. The middleware will be executed in the order you define them:

$router->group(['middleware' => 'auth'], function () use ($router) {
    $router->get('/', function () {
        // Uses Auth Middleware
    });

    $router->get('user/profile', function () {
        // Uses Auth Middleware
    });
});

Namespaces allow you to assign the same jsx namespace to a group of controllers. You can use the namespace parameter in the group attribute array to specify the namespace for all controllers within the group:

$router->group(['namespace' => 'Admin'], function() use ($router)
{
    // Using The "App\Http\Controllers\Admin" Namespace...

    $router->group(['namespace' => 'User'], function() use ($router) {
        // Using The "App\Http\Controllers\Admin\User" Namespace...
    });
});

The prefix group attribute can be used to prefix each route in the group with a given URI. For example, you may want to prefix all route URIs within the group with "admin":

$router->group(['prefix' => 'admin'], function () use ($router) {
    $router->get('users', function () {
        // Matches The "/admin/users" URL
    });
});

You can also use the prefix parameter to specify common parameters for your grouped routes:

$router->group(['prefix' => 'accounts/{accountId}'], function () use ($router) {
    $router->get('detail', function ($accountId) {
        // Matches The "/accounts/{accountId}/detail" URL
    });
});
