middlewares

HTTP middleware in JET provides a convenient mechanism for filtering HTTP requests entering your application. Middleware allows you to perform tasks such as authentication, CORS handling, logging, and more. In JET, all middleware should be stored in the app/Http/Middleware directory.

To create a new middleware, you can use the 

php jet jet:make-middleware Examplemddleware

command in your terminal. This will generate a new middleware class for you. Let's say we want to create a middleware that checks if the supplied age is greater than 200 and redirects the user if it's not. Here's an example of how the middleware class might look:

<?php

namespace Bcoded\Jet\Http\Middleware;

use Closure;
use Bcoded\Jet\Support\JetFacade as Jet;

class AgeCheckMiddleware
{
    public function handle($request, Closure $next)
    {
        if ($request->input('age') <= 200) {
            return redirect('home');
        }

        return $next($request);
    }
}

In this example, if the given age is less than or equal to 200, the middleware will return an HTTP redirect to the client. Otherwise, the request will be passed further into the application by calling the $next callback with the $request.

To register middleware globally for every HTTP request to your application, you can list the middleware class in the kernel.php file, located at app/Http/Middleware/Kernel.php. Add the middleware class to the $middleware array:

protected $middleware = [
    // Other middleware classes...
    \Bcoded\Jet\Http\Middleware\AgeCheckMiddleware::class,
];

If you want to assign middleware to specific routes, you can assign a shorthand key to the middleware class in the kernel.php file. Use the $routeMiddleware array for this:

protected $routeMiddleware = [
    // Other middleware mappings...
    'age.check' => \Bcoded\Jet\Http\Middleware\AgeCheckMiddleware::class,
];

Once the middleware has been defined in the kernel.php file, you can use the middleware key in the route options array to assign middleware to specific routes:

$router->get('admin/profile', ['middleware' => 'age.check', function () {
    // Route logic...
}]);

You can also assign multiple middleware to a route by passing an array:

$router->get('/', ['middleware' => ['first', 'second'], function () {
    // Route logic...
}]);

When attaching middleware to a controller route, you can use the uses key to specify the controller action as a string:

$router->get('admin/profile', [
    'middleware' => 'age.check',
    'uses' => 'AdminController@showProfile'
]);

Middleware can also receive additional custom parameters. For example, if your middleware needs to verify that the authenticated user has a given "role" before performing an action, you can define a parameter in the middleware's handle method. Here's an example:

public function handle($request, Closure $next, $role)
{
    if (! $request->user()->hasRole($role)) {
        // Redirect or perform some action...
    }

    return $next($request);
}

To specify middleware parameters when defining a route, separate the middleware name and parameters with a :. Multiple parameters should be delimited by commas:

$router->put('post/{id}', ['middleware' => 'role:editor', function ($id) {
    // Route logic...
}]);

Sometimes a middleware may need to perform additional work after the HTTP response has been sent

to the browser. To accomplish this, you can define a terminate method in the middleware. This method will be called by JET once the response has been sent. Here's an example:

public function terminate($request, $response)
{
    // Perform some post-response tasks...
}

To use terminable middleware, add it to the list of global middleware in the kernel.php file.

When using the terminate method in your middleware, JET will resolve a fresh instance of the middleware from the service container. This means that the handle and terminate methods will be called on separate instances of the middleware class.

If you want to use the same instance of the middleware for both the handle and terminate methods, you can register the middleware with the container using the container's singleton method.

Here's an example of how you can register the middleware as a singleton in JET's container:

  1. Open your JET application's app/providers/AppserviceProvider.php file.
  2. Locate the register method. It will look similar to this:
public function register()
{

}
  1. use the this to get the object of the app instance to register the middleware as a singleton: your code will look similar to this..
public function register()
{
    $this->$app->singleton(Bcoded\Jet\Http\Middleware\YourMiddleware::class);
}

By using the singleton method, JET will only resolve a single instance of the middleware, and the same instance will be used for both the handle and terminate methods.

Note that modifying the registration on the application container should be done cautiously, as it can have unintended consequences. If you're not familiar with how the container works, it's recommended to seek guidance or consult the JET support for more information. support@jetframework.info
