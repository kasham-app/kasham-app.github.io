Store Transaction Endpoint - POST Request

The store transaction endpoint allows the mobile app to record a payment transaction after a user has successfully made a payment to a third-party bank account. This endpoint is used in conjunction with a third-party payment API, where the mobile app handles the websocket connection to receive payment confirmation and then stores the transaction details. By sending a POST request to the following URL, payment transactions can be stored:

Endpoint: 

https://kashamng.com/api/v1/transaction/store

Required Headers​

To access this endpoint, you need to include the Authorization header with a valid bearer token. The bearer token can be obtained by authenticating the user through login or registration.

Headers:

Authorization: Bearer <your_token>

Required Parameters​

In order to successfully store a payment transaction, the following parameters are required in the request body. If any of these parameters are missing, the API will return an error response indicating the missing fields.

{
    "user_id": 1,
    "amount": "300"
}
  • user_id (integer): The unique identifier of the user who made the payment.
  • amount (string): The amount of the payment transaction.

Store Transaction Process​

Important Notice!

  1. Make sure you have obtained a valid bearer token by authenticating the user through login or registration.

  2. Send a POST request to the /api/v1/transaction/store endpoint with the required headers and parameters.

  3. The API will validate the request data. If the user_id and amount fields are missing, the API will return an error message.

  4. If the validation is successful, the mobile app should handle the websocket connection with the third-party payment API to receive the payment confirmation.

  5. After receiving the payment confirmation, the mobile app can then send a POST request to the /api/v1/transaction/store endpoint with the user's ID and the amount of the payment.

  6. The API will store the payment transaction in the database and update the user's wallet balance accordingly.

  7. After storing the transaction, the API will return a success response indicating that the payment was successful, along with the updated user's wallet information.

Store Transaction Response​

After successfully storing the payment transaction, the user balance will be incremented with the amount deposited and then the API will return the following response: (balance: the user balance table), (user: the user table) You can easily use the returned tables to perfom further validations and actions on the user.
