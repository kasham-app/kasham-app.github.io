controllers

The given code snippet represents a basic example of a controller class in a JET application. Controllers in JET are used to group related HTTP request handling logic into a class and are stored in the app/Http/Controllers directory.

Here's a breakdown of the provided code and some additional explanations:

namespace Bcoded\Jet\Http\Controllers;

use Bcoded\Jet\Support\Base\JetController;
use Illuminate\Http\Request;
use Illuminate\Http\Response;
use Bcoded\Jet\Support\JetFacade as Jet;
use League\Plates\Engine as Jetview;

class LoginController extends JetController
{
    // Controller methods...
}
  • The code begins with the definition of the namespace for the controller class.
  • The use statements import the necessary classes and dependencies for the controller.
  • The LoginController extends the JetController class, which is included with the default JET installation. This allows you to inherit functionality and methods provided by the JetController class.
  • Inside the LoginController class, you can define your controller methods to handle various HTTP requests.
public function show($id)
{
    return User::findOrFail($id);
}
  • The show method is an example of a controller method that retrieves a user based on the provided $id parameter.
  • It uses the findOrFail method of the User model to find the user with the given ID.
  • The retrieved user is then returned as a response.

Routing to the Controller: To route requests to the controller's action method, you can define routes in the routes/web.php file:

$router->get('user/{id}', 'UserController@show');
  • This route definition maps the GET request to the URL path user/{id} to the show method of the UserController.
  • The route parameter {id} will be passed to the show method as an argument.

Namespaces and Controller Routes: JET allows you to organize your controllers using PHP namespaces. If you have nested controllers in subdirectories, you can specify the class name relative to the root namespace (Bcoded\Jet\Http\Controllers):

$router->get('foo', 'Photos\PhotoController@method');
  • This example demonstrates routing to the method of the PhotoController located at Bcoded\Jet\Http\Controllers\Photos\PhotoController.

Naming Controller Routes: Similar to Closure routes, you can specify names for controller routes:

$router->get('foo', ['uses' => 'FooController@method', 'as' => 'name']);
  • The as key allows you to provide a name for the route.
  • You can use the route helper function to generate a URL to a named controller route.

Controller Middleware: You can assign middleware to the controller's routes using either the route definition or the controller's constructor:

$router->get('profile', [
    'middleware' => 'auth',
    'uses' => 'UserController@showProfile'
]);
  • In the above example, the 'auth' middleware is assigned to the 'profile' route.
  • Alternatively, you can specify middleware within the constructor of the controller itself.

Dependency Injection & Controllers: JET's service container allows for dependency injection in controllers. You can type-hint dependencies in the constructor or method parameters.

Constructor Injection:

public function __construct(Users $users)
{
    $this->users = $users;
}
  • The Users dependency is automatically resolved and injected into the UserController instance.

Method Injection:

public function store(Request $request)
{
    $name

 = $request->input('name');

    // ...
}
  • In this example, the Request instance is type-hinted in the store method and automatically injected.
  • You can access the request data using the injected $request object.

Route Parameter & Method Injection:

public function update(Request $request, $id)
{
    // ...
}
  • If your controller method expects input from both the request and a route parameter, you can list the dependencies accordingly.
  • The $id parameter represents the route parameter and follows the request dependency in the method signature.
