Update User Profile Endpoint - POST Request

The update user profile endpoint allows users to update their profile information within the Kasham application. By sending a POST request to the following URL, users can modify their account details:

Endpoint: 

https://kashamng.com/api/v1/update/user

Authentication​

To access the update user profile endpoint, the mobile app developer must include the Authorization header in the HTTP request. This header should contain the Bearer token obtained during either the user's login or registration process. This authentication ensures that only authorized users can update their profiles.

Example:
Authorization: Bearer eyJ0eXAiOiJKV1QiLCJhbGciOiJSUzI1NiJ9...

Request Parameters​

To successfully update a user's profile, the following parameters can be included in the request body:

  • isTagSet (string): Indicates whether the user's tag is set ("1" for yes, "0" for no).
  • email (string): The updated email address of the user.
  • password (string): The new password chosen by the user.
  • name (string): The updated display name of the user.
  • role (string): The user's updated role or permission level within the system.
  • account_type (string): The updated type of account the user holds (e.g., "merchant").
  • full_name (string): The updated full name of the user.
  • phone_number (string): The updated phone number of the user.
  • affiliate_id (string): The updated affiliate ID associated with the user.
  • referred_by (string): The Affiliate ID of the user who referred this user, if applicable.
  • sms_code (string): The updated SMS verification code, if applicable.
  • email_code (string): The updated email verification code, if applicable.
  • api_requests (string): The user's updated API request status ("1" for enabled, "0" for disabled).
  • is_pro (string): Indicates whether the user has a pro membership. user ("1" for yes, "0" for no).
  • stateof_origin (string): The updated state of origin of the user.
  • reasonfor_kasham (string): The updated reason for using Kasham.
  • pin_code (string): The updated PIN code of the user for login-in to their account.
  • selfie_upload (file): The updated selfie image of the user (PNG or JPG format).
  • document_upload (file): The updated document image of the user (PNG or JPG format).
  • bvn_number (string): The updated Bank Verification Number (BVN) of the user.
  • kasham_tag (string): The updated Kasham tag of the user.

Validation and File Upload​

Before updating the user profile, the mobile app developer should ensure that the uploaded files meet the required criteria. For both the selfie_upload and document_upload files:

  • Make sure that the file format is PNG or JPG.
Handling The Uploaded files
  • Laravel Generate a unique filename with a timestamp and extension.
  • Laravel Move the file to the appropriate directory (uploads/selfie or uploads/documents).
  • Laravel Update the corresponding attribute in the model (selfie_upload or document_upload).
  • Laravel validates the file mimes to make sure it corresponds with the (required mimes in the application)

Update Process​

When handling the update process, Laravel will update the user's profile based on the provided parameters. If certain fields are not included in the request, the existing values will remain unchanged.

The update process involves the following steps:

  1. Get the authenticated user using $request->user().
  2. Prepare the onboarding data based on the provided onboarding parameters.
  3. Update the user's attributes based on the request parameters, including name, email, password, role, account type, and more.
  4. Validate and process the uploaded selfie image and document image, updating the respective attributes.
  5. Update the user's Bank Verification Number (BVN) and Kasham tag if provided.
  6. Save the onboarding data and user profile changes to the database using $user->save().
  7. Respond with a JSON message confirming the successful account update.

Response​

After a successful update, the API will return the following response:

{
    "message": "Account updated successfully."
}
